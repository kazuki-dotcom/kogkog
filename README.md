# Dead Man's Switch (デッドマンズ・スイッチ)

> **「隠したい秘密は闇に葬り、遺される人には実利だけを届ける」**
> B2B福利厚生型 デッドマンズスイッチ & プライバシーパージシステム

## 🚀 プロジェクト概要
`Dead Man's Switch` は、ユーザーの不慮の事故・死亡時に、**PCやスマホの全情報を遺族に見られないように防衛（パージ）しつつ、行政・財産手続きに必要な最小限の情報（結果）だけを抽出して遺族に届ける**、B2B（福利厚生）向けのSaaSプラットフォームです。

また、企業向け機能として、**死亡した社員の「預託業務品の回収」「社用アカウントのセキュリティ対応」「バックオフィス（人事・労務・法務）の死亡退職手続き」を一元化して自動・標準化するコーポレートガバナンス機能**も内包しています。

### 🟥 L1/L2: 自爆・パージレイヤー (ユーザーの尊厳を死守)
会社からのトリガー発動と同時に、バックグラウンドで実行されます。
* **Google API連携:** 検索履歴、YouTube視聴履歴、Googleマップの移動履歴を完全消去。
* **Gmail / メールフィルタリング:** 事前に登録した「NGキーワード（例: 消費者金融名、病院・クリニック、マッチングアプリ）」を含むメールをゴミ箱からも完全削除。
* **SNS API連携:** LINEやその他SNSのアカウント連携を解除、または退会リクエストを送信し、トーク履歴ごと消滅させる。
* **★ソフトデリート（復元プロトコル）の担保:** * 誤操作や「本人の奇跡的な生存・回復」に備え、削除データや解除した連携のインデックスは、システム側の暗号化セキュアストレージに**30日間一時隔離（ソフトデリート）**されます。
    * 30日以内に本人の有効な認証（または企業とシステム運営者の二重承認）がない限り、31日目に物理的に完全削除され、世界から消滅します。

### 🟩 L3: エッセンシャルレイヤー (遺族への実利提供)
遺族には「探索権限」を与えず、**「整理済みの結果（PDF）」をメールでステップ配信**します。
* **資産手続きのミニマル化:** 銀行口座名・支店名、保険証券番号、マイナスの財産（借入残高）のみをリスト化して送付。
* **サブスク自動パージ:** クレジットカード明細で遺族を困惑させる「嗜好性の高いサブスク」の解約URLとIDのみを抽出し、遺族に代行処理させる。
* **★シナリオ配信のタイムライン制御 & プレビュー機能:** * 企業の管理画面から、遺族に届く予定のメール内容（PDFの中身を含む）をすべて**事前にプレビュー確認**できます。
    * 遺族の状況（葬儀の進捗や精神状態）に合わせ、管理者はボタン一つで「次のメール配信を今すぐ実行する（前倒し）」、または「一時停止する」というコントロールが可能です。

### 🟦 企業向け：死亡退職対応・ガバナンスレイヤー
トリガー発動と同時に、情報システム部門・総務・人事労務部門のタスクを自動化・標準化します。

* **預託業務品の回収・セキュリティ対応:**
    * **社用アカウントの「エスクロー閲覧権限」の付与:** 「業務引き継ぎ」や「取引先との連絡整理」のため、企業アカウント（M365 / Google Workspace等）に対する一時的な管理者閲覧プロキシを生成。管理画面から故人のメールやチャットログを検索・閲覧可能にします。
    * **業務ファイルの自動サルベージ:** デバイスを完全ロックする直前の数分間で、ローカルPC内にある最新の重要ファイルを企業の共有ドライブ（Google Drive / SharePoint等）へ自動バックアップ（救出）します。
    * **備品返却指示書の自動生成:** 貸与備品（PC、スマホ、社章、鍵等）のリストと遺族用の返却手順案内を自動で作成し、遺族宛ての手続きメールに自動挿入。
* **社外引き継ぎアシスタント:**
    * 直近1ヶ月でやり取りの頻度が高かった社外の主要取引先アドレスを自動抽出し、訃報および後任案内のメール下書き（テンプレート）を自動生成。
* **バックオフィス手続きマニュアル（標準タスク ＋ 自由設定カスタム）:**
    従業員の死亡に伴い、企業側が対応すべき法的・事務的手続きをダッシュボード上でタスク管理（To-Do）化します。以下の主要項目を標準搭載し、企業ごとに自由に項目を追加・編集（その他企業カスタマイズ）できます。
    
    | 分類 | 標準搭載タスク（主要項目リスト） | 対応期限（目安） |
    | :--- | :--- | :--- |
    | **給与・支払停止** | 死亡日時点での給与計算の停止（誤振込防止）、当月分給与・未払賃金の計算 | 即時 |
    | **社会保険** | 健康保険・厚生年金保険 被保険者資格喪失届の提出、健康保険証の回収 | 死亡後5日以内 |
    | **雇用保険** | 雇用保険 被保険者資格喪失届、離職証明書の提出 | 死亡日の翌日から10日以内 |
    | **税金** | 給与所得者の死亡退職届出書の提出（住民税の特別徴収から普通徴収への切替） | 翌月10日まで |
    | **社内精算** | 経費精算（未精算の出張費・立替金など）の確定・相殺 | 企業ルールに準ずる |
    | **福利厚生** | 企業独自の退職金・弔慰金の支給手続き、社葬・供花・弔電の手配 | 企業ルールに準ずる |
    | **カスタム** | **【その他企業カスタマイズ】（各企業が自由にタスクを追加・設定可能）** | 自由設定 |

## ⚡️ サービスのライフサイクル（トリガー・フロー）

### 1. 企業によるユーザー招待 & 生前設定
* **アクティベーションコードの発行:** 企業管理者が社員を登録すると、システムから各社員へ「アクティベート専用の個別の使い捨てトークンURL」がセキュアに発行される。
* **User Sideの設定:** 社員は届いたコードを入力してアカウントを有効化。Google/SNS等のOAuth認証（履歴削除権限の認可）を行い、遺族に「見せて良い情報」と「消したいキーワード」を設定。

### 2. トリガー発動 (B2B / Company Side)
* 不慮の事故によりユーザーが死亡。遺族から会社（総務・人事）へ連絡が入る。
* **人事担当者が管理画面から、当該社員の「死亡確定ボタン」を押す。**

### 3. パージ & リリース & 企業側タスク発動 (System Side)
* **即時（数秒以内）:** 社員のプライベートな検索履歴・特定メール・SNSがソフトデリート（一時隔離）され、見かけ上完全消滅。同時に、企業側では「ローカルデータの自動サルベージ」が走り、社用アカウントへの管理者閲覧権限が有効化。
* **バックオフィス画面:** 死亡退職に伴う「人事労務の標準タスク（給与停止・社保手続き等）」および「企業カスタムタスク」のTo-Doリストが生成され、担当者に通知。取引先への引き継ぎメール下書きも自動生成される。
* **1日後:** 遺族のアドレス宛に、第一弾メール「葬儀・会社への預託品返却手順に関する手続きPDF」を送信。**（※管理者がプレビュー確認し、手動で送信タイミングを操作可能）**
* **3日後:** 第二弾メール「金融口座・保険・解約すべきサブスクリストPDF」を送信。

## 🎯 開発スコープ (MVP / 最小構成)

Vibe Codingの第一段階として、以下のコンポーネントをミニマムで実装します。

- [ ] **Auth / ユーザー管理:** 企業アカウント（Admin）と従業員アカウント（User）の2階層。
- [ ] **企業管理者画面:** - 従業員の登録および個別アクティベーションコード（トークン）の発行・管理。
    - 各従業員に紐づく「貸与している備品」の入力フォーム。
    - **【New】** 死亡退職手続き用の標準To-Doリスト（給与停止、社保等）の進捗管理 ＆ 企業カスタムタスクの追加・編集UI。
    - 遺族向けステップメールの「プレビュー機能」および「配信加速 / 一時停止」コントロールパネル。
    - トリガー発動後の「社用アカウント・ログ閲覧用ダミープロキシUI」。
- [ ] **従業員ダッシュボード:**
    - 企業から支給されたコードによる初回ログイン（アクティベート）処理。
    - 遺族に伝える「銀行・保険」のテキスト入力フォーム。
    - Gmail連携（OAuth）および「消したいキーワード」の設定機能。
- [ ] **バックエンド処理 (Trigger発生時):**
    - 指定キーワードを含むGmailの「一時隔離（復元可能領域へのムーブ）」スクリプトの実行。
    - 企業向けの「備品返却指示」を含んだ、遺族への手続き案内PDFの自動生成とメール配信システム。
